Site da Oncase

Ruby e Rails

A versão do Ruby utilizada é a 2.5.0.

A versão do Rails utilizada é a 5.1.6.

Setup local

Instale o Ruby, o Rails e o PostgreSQL para o seu sistema operacional seguindo o guia do gorails.com.

Clone este repositório.

$ git clone https://github.com/rafael-fernandes/site-oncase

Instale as gems:

$ bundle install

Crie o banco de dados e execute as migrações:

$ rails db:create
$ rails db:migrate

Preencha o banco de dados para utilizar o CMS utilizando:

$ rails db:seed

Crie o usuário para acessar o CMS:

$ rails console
>> User.create(email: "email@example.com", password: "password")

Inicie o servidor:

$ rails server

Acesse o site pelo endereço localhost:3000 e o CMS pelo endereço localhost:3000/members.

Setup na Dreamhost

Acesse o servidor remotamente utilizando SSH:

$ ssh oncasec@oncase.com.br

Insira a senha para poder utilizar a linha de comando.

Tenha certeza que você está no diretório home do usuário oncasec:

$ cd ~

Instale as chaves públicas do RVM:

$ gpg --keyserver hkp://keys.gnupg.net --recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3 7D2BAF1CF37B13E2069D6956105BD0E739499BDB

Instale a última versão estável do RVM:

$ curl -sSL https://get.rvm.io | bash -s

Execute para remover o seu GEM_HOME:

$ unset GEM_HOME

Execute para criar sua nova instalação do RVM:

$ source ~/.rvm/scripts/rvm

Desabilite as autolibs:

$ rvm autolibs disable

Cheque se o RVM está instalado e funcionando:

$ rvm --version
rvm 1.29.2 (latest) by Michal Papis, Piotr Kuczynski, Wayne E. Seguin [https://rvm.io/]

Instale a versão do Ruby:

$ rvm install 2.5.0

Clone o repositório do site para oncase.com.br:

$ git clone https://github.com/rafael-fernandes/site-oncase oncase.com.br

Vá para a pasta do site:

$ cd oncase.com.br

Utilizando a versão instalada do Ruby:

$ rvm use 2.5.0

Crie um gemset:

$ rvm gemset create oncase

E utilizando este gemset:

$ rvm gemset use oncase

Instale o bundler:

$ gem install bundler

Crie um arquivo .htaccess na pasta do site:

$ touch .htaccess

E edite ele com o seguinte conteúdo:

PassengerRuby /home/oncasec/.rvm/wrappers/ruby-2.5.0@oncase/ruby
PassengerAppEnv production
RewriteEngine On
RewriteCond %{HTTPS} !=on
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

Instale as gems:

$ bundle install --without development test

Crie uma chave secret:

$ rake secret
2y63t434dfsdfds61ec1829ffa40985cdf10db17d65c7af7sfdsfsabdb6e7c28812a74efe901b76e598e50sdfsdfsdfsdfs75ba2e48

E adicione ela ao arquivo .bashrc na home do usuário oncasec:

export SECRET_KEY_BASE='2y63t434dfsdfds61ec1829ffa40985cdf10db17d65c7af7sfdsfsabdb6e7c28812a74efe901b76e598e50sdfsdfsdfsdfs75ba2e4'

Volte à pasta do site:

$ cd oncase.com.br

Instale as migrações do banco de dados:

$ RAILS_ENV=production rails db:migrate

E preencha o banco de dados com informações do CMS:

$ RAILS_ENV=production rails db:seed

Crie o usuário para acessar o CMS:

$ RAILS_ENV=production rails console
>> User.create(email: "email@example.com", password: "password")

Reinicie a aplicação:

$ touch tmp/restart.txt
